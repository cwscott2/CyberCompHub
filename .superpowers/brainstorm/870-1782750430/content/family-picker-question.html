Where should the family picker live in the wizard?

The family picker filters controls to a single security domain (e.g. "Access Control"). Here are three placement options.

A

Step 3 — Between Template and Scope

Framework → Template → Family → Scope/Controls → Generate → Export

Family picker appears as its own dedicated step. Controls list is pre-filtered when the user arrives at Scope. Clean separation of concerns. Adds one step to the flow.

B

Inline on the Template step

Framework → Template + Family → Scope/Controls → Generate → Export

After picking a procedure/gap template, a family dropdown appears inline on the same card. One fewer step. Family selector only appears for template types that require it (procedure, gap_assessment).

C

Top of Scope step

Framework → Template → Scope (family first, then controls) → Generate → Export

No new step added. Family picker sits at the top of the existing Scope card. Selecting a family dynamically filters the controls list below it. Simplest diff — only one card changes.
